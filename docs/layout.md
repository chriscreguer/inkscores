# Layout

Display: **800×480 landscape**, E Ink Spectra 6 (E6) ePaper. Slow refresh, low
power — static screens only, no animation or scrolling.

## MVP layout (MLB season)

```
┌────────────────────────────────────────────────────────────┐
│ SPORTS DASHBOARD                         Updated 8:05 AM    │  header ~44px
├──────────────────────────────┬─────────────────────────────┤
│ Tigers                       │ Cubs                        │  team cards
│ W 5-3 vs CLE                 │ L 4-2 vs STL                │  ~120px
│ Next: Today 6:40 vs MIN      │ Next: Today 1:20 vs PIT     │
│ 42-34 | AL Central: 2nd      │ 39-37 | NL Central: 3rd     │
├──────────────────────────────┼─────────────────────────────┤
│ AL CENTRAL                   │ NL CENTRAL                  │  standings
│ 1 CLE  44-31   -             │ 1 MIL  43-32   -            │  fill remaining
│ 2 DET  42-34   2.5           │ 2 STL  41-35   2.5          │
│ 3 KC   38-38   6.5           │ 3 CHC  39-37   4.5          │
│ 4 MIN  36-40   8.5           │ 4 CIN  36-40   7.5          │
│ 5 CWS  28-48   16.5          │ 5 PIT  31-45   12.5         │
└──────────────────────────────┴─────────────────────────────┘
```

The renderer partitions sections by type and places the first two `teamCard`s
in the top row and the first two `standings` tables in the bottom area. A
`message` section fills the content area when there are no cards (offseason), or
occupies a spare column otherwise.

## Constants

Defined in `firmware/src/dashboard_types.h`:

| Constant | Value |
|----------|-------|
| outer margin | 16 px |
| section gap | 10 px |
| header height | 44 px |
| team card height | 120 px |

## Type sizes

The firmware uses the Adafruit GFX built-in font scaled with `setTextSize` to
avoid bundling font files:

| Element | size |
|---------|------|
| Header / team names | 3 |
| Body (last/next/standing) | 2 |
| Standings rows | 2 |

## Colour (Spectra 6)

The panel has six inks: **black, white, red, yellow, green, blue** (no orange —
`orange` accents map to red). Colour is used for *meaning*, never decoration,
and never as the only cue (the manifesto rule still holds):

- **Team logo** anchors the top-left of each card (no left stripe). The
  contract carries two forms:
  - `logoUrl` — the full-colour logo from ESPN's CDN. Image-capable clients
    (the browser preview) render this directly.
  - `badge` — a 1-3 char monogram fallback for any team without a baked logo.
- The **firmware** ships real logos as 44×44 bitmaps pre-quantized to the six
  inks (`firmware/src/logos.h`, generated by `tools/gen-logos.py`). It blits the
  logo by team key and falls back to the monogram badge when none exists. The
  panel can't decode a PNG at runtime, so the conversion happens at build time.
- **Team name** is black (the logo carries the colour, keeping the card calm).
- **W/L result** on the last-game line is green (W) / red (L); the letter still
  reads W or L, so colour is reinforcement only.
- **LIVE badge** is a solid red chip with white text.
- **Watched standings row** gets a yellow highlighter fill with black text (the
  one ink light enough to keep text readable), and its team name in the accent
  colour. The fill means the highlight survives a degraded/monochrome render.
- No gradients, shadows, transparency, or large dark fills (refresh artifacts).
  Yellow is light, so a full-row yellow fill is safe.

## Text fitting

`fitText(value, maxChars)` truncates with an ellipsis. The MVP does no complex
word wrapping — the backend keeps strings short (concise labels, team
abbreviations) so single-line truncation is enough.
